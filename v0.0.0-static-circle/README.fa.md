# راهنمای پیاده‌سازی معماری ECS - نسخه v0.0.0 (دایره ثابت)

English Version: For the English documentation, please refer to README.md.

این زیرپروژه نشان‌دهنده بنیادی‌ترین لایه از سیستم مدیریت داده‌محور Entity Component System (ECS) ما در زبان C++ است. هدف این نسخه، نمایش تفکیک کامل داده‌ها از منطق برنامه، تنها از طریق رندر کردن یک دایره قرمز ثابت روی صفحه با استفاده از کتابخانه Raylib است.

---

## ساختار معماری پروژه

کدهای این نسخه کاملاً منطبق بر اصول خالص معماری ECS پیاده‌سازی شده‌اند:

* **موجودیت (Entity):** یک شناسه عددی خالص (`uint32_t`) است. این شناسه هیچ وزن ساختاری، رفتار یا پیش‌فرضی درباره کامپوننت‌ها ندارد.
* **مؤلفه‌ها (Components):** ساختارهای داده‌ای خالص (`Position` و `RenderComponent`) که فقط داده‌های خام را به صورت متوالی در حافظه نگه می‌دارند و فاقد هرگونه منطق اجرایی هستند.
* **پایگاه داده (Registry):** یک سیستم ثبت مرکزی کامپوننت‌ها با چیدمان موازی آرایه‌ها (SoA). تخصیص حافظه در این بخش کاملاً پویا (On-demand) است و تغییر سایز خودکار آرایه‌ها تعبیه شده تا از کرش‌های خروج از محدوده حافظه (Out of Bounds) جلوگیری شود.
* **سیستم‌ها (Systems):** توابع جهانی مستقل (`RenderSystem`) که هیچ داده‌ای درون خود ذخیره نمی‌کنند. آن‌ها آرایه‌ها را بر اساس ماسک‌های حضور کامپوننت فیلتر کرده و انتیتی‌های واجد شرایط را رندر می‌کنند.

---

## پیش‌نیازها

* یک کامپایلر با پشتیبانی از C++17 یا بالاتر (MSVC, GCC یا Clang)
* ابزار مدیریت بیلد CMake (نسخه 3.16 یا بالاتر)
* ادیتور VS Code (پیشنهاد شده)

## راهنمای گام‌به‌گام اجرا و رفع خطاهای محیطی

برای ساخت و اجرای این نسخه، نباید دستورات بیلد را در پوشه اصلی و ریشه ریپازیتوری اجرا کنید. از یکی از دو روش زیر استفاده کنید:

### روش اول: استفاده از افزونه CMake Tools در VS Code (روش پیشنهادی)

این روش به شما اجازه می‌دهد بدون درگیر شدن با خطاهای مسیر ترمینال، پروژه را از طریق رابط کاربری ادیتور کامپایل و اجرا کنید.

1. **جداسازی پوشه:** نرم‌افزار VS Code را باز کنید. از منوی بالا به مسیر `File > Open Folder` بروید و **فقط** پوشه `v0.0.0-static-circle` را باز کنید. کل پروژه را یک‌جا باز نکنید.
2. **انتخاب کامپایلر (Kit):** کلیدهای `Ctrl + Shift + P` را بزنید تا منوی دستورات باز شود. عبارت `CMake: Select a Kit` را سرچ کرده و کامپایلر نصب شده روی سیستم خود را انتخاب کنید (مانند Visual Studio Community Release - amd64 یا GCC).
3. **پیکربندی پروژه:** دوباره `Ctrl + Shift + P` را زده، عبارت `CMake: Configure` را تایپ و اجرا کنید. در این مرحله CMake فایل کانفیگ را خوانده و نسخه Raylib 6.0 را به صورت خودکار از گیت‌هاب درون پوشه بیلد دانلود می‌کند. منتظر بمانید تا پیغام `Configuring done` ظاهر شود.
4. **کامپایل و اجرا:** در نوار وضعیت پایین VS Code، روی دکمه **Build** کلیک کنید تا پروژه کامپایل شود. سپس روی دکمه **Play** (مثلث کوچک) کلیک کنید تا برنامه اجرا شود.

### روش دوم: استفاده از خط فرمان (Terminal CLI)

اگر دستور `cmake ..` را در ترمینال استاندارد ویندوز زدید و با خطای *"The term 'cmake' is not recognized..."* مواجه شدید، به این معناست که مسیر ابزار CMake در متغیرهای سیستم (PATH) شما ثبت نشده است. برای حل این مشکل از یکی از دو راه‌حل زیر استفاده کنید:

* **راه‌حل اول (محیط ویژوال استودیو):** از منوی استارت ویندوز، ابزار **Developer PowerShell for VS** را باز کنید، با دستور `cd` به پوشه build پروژه بروید و دستورات را آنجا تایپ کنید.
* **راه‌حل دوم (ترمینیال اختصاصی افزونه):** در VS Code کلیدهای `Ctrl + Shift + P` را بزنید و عبارت `CMake: Open CMake Tools Extension Terminal` را اجرا کنید. این ترمینال تمام مسیرها را به صورت پیش‌فرض درون خود دارد.

پس از ورود به محیط ترمینال درست، دستورات زیر را به ترتیب اجرا کنید:

```bash
# ۱. ایجاد پوشه بیلد و ورود به آن
mkdir build
cd build

# ۲. پیکربندی و شروع دانلود خودکار کتابخانه Raylib 6.0
cmake ..

# ۳. کامپایل نهایی پروژه
cmake --build .

# ۴. اجرای فایل باینری خروجی
# در ویندوز (کامپایلر پیش‌فرض MSVC):
.\Debug\ecs_static_circle.exe
# در لینوکس و مک:
./ecs_static_circle

## تمرین

برای سنجش درک خود از این ساختار اولیه ECS، چالش زیر را پیاده‌سازی کنید:
* **چالش:** یک انتیتی (موجودیت) دوم بسازید که یک دایره **آبی** با شعاع **۳۰** پیکسل را در موقعیت دیگری از صفحه (مثلاً `x: 200.0f, y: 150.0f`) رندر کند.
* **هدف تمرین:** دقت کنید که چگونه بدون دست زدن به کدهای داخلی `RenderSystem` یا تعریف یک کلاس جدید، تنها با ساخت یک Entity ID جدید و تزریق مؤلفه‌ها، یک شیء کاملاً جدید روی صفحه ظاهر می‌شود.
