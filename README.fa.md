# راهنمای جامع پیاده‌سازی معماری داده‌محور ECS در C++

English Version: For the English documentation, please refer to README.md.

---

## هدف این پروژه چیست؟

هدف این ریپازیتوری، آموزش و نمایش گام‌به‌گام پیاده‌سازی یک سیستم ECS است. ما پروژه را از یک ساختار بسیار ساده و ابتدایی (نسخه v0.1.0) شروع می‌کنیم و به مرور زمان ویژگی‌ها و مفاهیم جدیدی را در قالب نسخه‌های بعدی به آن اضافه می‌کنیم تا روند تکامل تدریجی یک سیستم دیتامحور را در عمل مشاهده کنیم.

---

## سیستم ECS چیست؟

معماری ECS (مخفف Entity Component System) یک الگوی طراحی نرم‌افزار است که به طور ویژه در توسعه بازی‌های ویدیویی و شبیه‌سازی‌های کامپیوتری استفاده می‌شود. 

این معماری بر پایه اصول طراحی داده‌محور (Data-Oriented Design) استوار است. برخلاف رویکرد شیءگرایی سنتی (OOP) که داده‌ها و رفتارها را در کلاس‌ها با هم ترکیب می‌کند، در ECS داده‌های خام کاملاً از منطق و پردازش برنامه تفکیک می‌شوند. این کار به ساختاریافته‌تر شدن پروژه‌ها و مدیریت راحت‌تر ویژگی‌های بازی کمک می‌کند.

مطالعه بیشتر: از آنجا که این پروژه روی پیاده‌سازی عملی تمرکز دارد، برای درک عمیق‌تر تئوری‌های پشت صحنه و مقایسه مفصل بین شیءگرایی (OOP) و طراحی داده‌محور (DOD)، پیشنهاد می‌شود مقالات تخصصی مربوط به این الگوها را مطالعه کنید.

## پیش‌نیازها و نحوه اجرا

این پروژه به صورت نسخه به نسخه توسعه می‌یابد. کدهای هر نسخه به صورت یک پروژه مستقل درون پوشه مخصوص به خود قرار دارند. برای اجرای هر کدام از نسخه‌ها به ابزارهای زیر نیاز دارید:

* یک کامپایلر با پشتیبانی از C++17 یا بالاتر (MSVC, GCC, یا Clang)
* ابزار مدیریت بیلد CMake (نسخه 3.16 به بالا)
* کتابخانه گرافیکی Raylib (مدیریت وابستگی‌ها به صورت خودکار از طریق CMake انجام می‌شود)

### دستورات کامپایل در ترمینال:

برای اجرای هر نسخه، ابتدا ترمینال را در پوشه اصلی پروژه باز کرده و دستورات زیر را وارد کنید (به جای v0.1.0-basic-collision نام پوشه نسخه مورد نظرتان را بنویسید):

```bash
# ۱. رفتن به پوشه نسخه مورد نظر
cd v0.1.0-basic-collision

# ۲. ایجاد پوشه بیلد، پیکربندی و کامپایل پروژه
mkdir build && cd build
cmake ..
cmake --build .
