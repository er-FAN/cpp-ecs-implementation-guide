## پیش‌نیازهای مفهومی

برای درک بهتر معماری ECS، پیشنهاد می‌شود قبل از شروع پیاده‌سازی، با مفاهیم پایه زیر آشنا شوید:

* برنامه‌نویسی شی‌گرا (Object-Oriented Programming یا OOP)
* طراحی داده‌محور (Data-Oriented Design یا DOD)
* مفاهیم مدیریت حافظه، ساختار داده‌ها و نحوه دسترسی به داده‌ها در C++

آشنایی با این مفاهیم باعث می‌شود دلیل وجود هر بخش از ECS و تفاوت آن با معماری‌های سنتی بهتر درک شود.

---

## ECS چه زمانی استفاده می‌شود؟

ECS معمولاً زمانی مورد استفاده قرار می‌گیرد که یک برنامه نیاز به مدیریت تعداد زیادی موجودیت مشابه داشته باشد و پردازش داده‌ها اهمیت زیادی پیدا کند.

برخی از کاربردهای رایج ECS:

* موتورهای بازی و سیستم‌های گیم‌پلی
* شبیه‌سازی‌های کامپیوتری
* برنامه‌هایی با تعداد زیادی آبجکت یا موجودیت فعال
* سیستم‌هایی که نیاز به پردازش سریع و بهینه داده‌ها دارند

برای مثال، در یک بازی ممکن است هزاران موجودیت هم‌زمان وجود داشته باشند؛ مانند بازیکن، دشمنان، گلوله‌ها، اشیای محیط و افکت‌ها. هر کدام از این موجودیت‌ها می‌توانند ترکیب متفاوتی از ویژگی‌ها داشته باشند.

به جای ساختن کلاس‌های پیچیده برای هر نوع موجودیت، در ECS می‌توان داده‌ها را به Componentهای جداگانه تقسیم کرد:

* Position برای موقعیت
* Velocity برای سرعت
* Health برای سلامتی
* RenderData برای اطلاعات نمایش

سپس Systemهای مختلف روی این داده‌ها پردازش انجام می‌دهند:

* Movement System برای حرکت موجودیت‌ها
* Rendering System برای نمایش
* Physics System برای شبیه‌سازی فیزیک

## ECS در موتورهای بازی

ایده‌های مرتبط با ECS در بسیاری از موتورهای بازی مدرن و ابزارهای توسعه بازی مورد استفاده قرار گرفته‌اند.

برای مثال، در موتور بازی Unity یک معماری داده‌محور با نام **Unity DOTS (Data-Oriented Technology Stack)** معرفی شده است که شامل سیستم‌هایی مانند Entity Component System، Job System و Burst Compiler می‌شود. هدف این معماری افزایش کارایی در پروژه‌هایی با تعداد زیادی موجودیت است.

در موتور Unreal Engine نیز اگرچه معماری اصلی موتور مبتنی بر کلاس‌ها و Actorها است، اما در بخش‌هایی از آن از تکنیک‌های داده‌محور و سیستم‌هایی با اهداف مشابه برای بهبود عملکرد استفاده شده است.

علاوه بر موتورهای بزرگ، بسیاری از موتورهای کوچک‌تر و کتابخانه‌های بازی‌سازی نیز ECS را به دلیل سادگی، انعطاف‌پذیری و کارایی آن انتخاب کرده‌اند.

## ساخت و اجرای پروژه

ابتدا مخزن پروژه را دریافت کنید و سپس وارد پوشه نسخه مورد نظر شوید.

برای مثال:

```bash
cd v0.0.0-project-name

//سپس یک پوشه برای فایل‌های ساخته‌شده ایجاد کنید:
mkdir build
cd build

//در این مرحله، CMake فایل‌های لازم برای سیستم ساخت پروژه را تولید می‌کند:
cmake ..

//پس از پایان موفقیت‌آمیز مرحله پیکربندی، پروژه را کامپایل کنید:
cmake --build .
```

پس از پایان کامپایل، فایل اجرایی ساخته‌شده در پوشه build قرار خواهد گرفت.
